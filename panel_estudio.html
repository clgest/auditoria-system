Buscar trámite / solicitud existente

Use este buscador para continuar un trámite ya generado: agregar documentación, enviar comunicación, cargar Alta Temprana o cerrar. Busque por Nº formal, ID sistema, CUIT, razón social, CUIL, postulante, mail, estado o beneficio. Al seleccionar, el sistema toma automáticamente ID, CUIT, CUIL, mail y datos ya cargados.
Seleccione una solicitud para usar sus datos en los módulos del Panel.

Beneficio aplicable

Administre el catálogo de beneficios. La vigencia y estado normativo condicionan si se puede generar una solicitud limpia o si requiere revisión.
Uso operativo: para una nueva solicitud solo seleccione un beneficio del catálogo. Vigencia, fin de vigencia, requisitos, documentación y advertencias se autocompletan desde la base. No deben recargarse por cada alta.

Crear solicitud y enviar link

Beneficio: la solicitud toma el beneficio seleccionado desde el catálogo. No vuelva a cargar vigencia, fin de vigencia ni requisitos en cada formulario.
Genere token/link y envíelo al empleador. El empleador reenvía el link al postulante.

Lote de altas simultáneas

Use lote cuando un empleador solicita dos o más altas. Cada postulante tiene su propio token/link.
Use esta opción cuando un mismo empleador solicite dos o más altas. Se genera un token/link individual por postulante, dentro de un mismo lote.

Canal formal de comunicación con empleador, postulante o terceros. Todo envío o gestión debe quedar registrado.

Alta Temprana ARCA

Suba la constancia de Alta Temprana emitida por ARCA. Luego envíela al empleador con instrucciones de firma, entrega de copia al trabajador, archivo físico en legajo y carga posterior de la copia firmada.
Hito crítico del cierre operativo. La constancia de Alta Temprana debe enviarse al empleador y luego debe recibirse la copia firmada.

Cierre

Cierre solo cuando estén completos documentos, PDF, Alta Temprana y copia firmada cuando corresponda.
